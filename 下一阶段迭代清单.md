# RuseDB 下一阶段迭代清单（V1.0.1 迭代）

## 目标

- 在现有 v0.1 基础上，从“可用”提升到“更可靠、可扩展、可维护”。
- 优先补齐核心数据库语义（约束、DDL 演进、并发与可观测性）。

## 里程碑总览

- Milestone A：约束系统（主键/唯一键/外键）
- Milestone B：DDL 演进能力（ALTER/RENAME）
- Milestone C：查询能力增强（更多谓词与 JOIN 语义）
- Milestone D：事务并发与恢复增强
- Milestone E：优化器与性能可观测
- Milestone F：运维能力与生态接入

---

## Milestone A：约束系统（P0）

### A1. PRIMARY KEY

- [X] SQL 语法支持：列级和表级主键定义
- [X] Catalog 存储主键信息
- [X] 插入/更新时校验主键唯一性
- [X] 主键列默认 `NOT NULL`
- [X] 错误码与错误信息统一（如 duplicate key）
- [X] 单元测试 + 集成测试（插入冲突、更新冲突、空值场景）

### A2. UNIQUE

- [X] SQL 语法支持：列级/表级 UNIQUE
- [X] 建立对应唯一索引或唯一性检查路径
- [X] 插入/更新冲突检测
- [X] 测试覆盖（多列唯一、重复值、事务内冲突）

### A3. FOREIGN KEY

- [X] SQL 语法支持：`FOREIGN KEY ... REFERENCES ...`
- [X] Catalog 持久化外键元数据
- [X] 插入/更新时引用完整性校验
- [X] 删除/更新父表行为（先支持 `RESTRICT`）
- [X] 跨表事务一致性测试

### A 验收标准

- [X] `CREATE TABLE` 可定义 PK/UNIQUE/FK
- [X] 违反约束时返回稳定可读错误
- [X] 重启后约束元数据仍生效

---

## Milestone B：DDL 演进能力（P1）

### B1. ALTER TABLE

- [X] `ALTER TABLE ... ADD COLUMN`
- [X] `ALTER TABLE ... DROP COLUMN`（受限规则先行）
- [X] `ALTER TABLE ... ALTER COLUMN`（后续可扩）

### B2. RENAME

- [X] `RENAME TABLE`
- [X] `RENAME COLUMN`
- [X] 关联元数据同步（索引、约束引用）

### B3. 安全与兼容

- [X] DDL 事务边界与失败回滚策略
- [X] 向后兼容测试（旧数据目录升级）

### B 验收标准

- [X] 基本 schema 演进可在不重建库的前提下完成
- [X] DDL 异常不破坏 catalog 一致性

## Milestone C：查询能力增强（P1）

### C1. 谓词与表达式

- [X] `IN`
- [X] `LIKE`
- [X] `BETWEEN`
- [X] `IS NULL / IS NOT NULL`

### C2. JOIN 与聚合

- [X] `LEFT JOIN`
- [X] `HAVING`
- [X] 聚合与分组边界场景测试

### C3. 子查询（分阶段）

- [X] 第一阶段：`IN (SELECT ...)`
- [X] 第二阶段：标量子查询

### C 验收标准

- [X] 常见业务 SQL 不需要大量在应用层改写
- [X] SQL 解析报错位置准确、信息可读

---

## Milestone D：事务并发与恢复增强（P1）

### D1. 并发控制

- [X] 明确隔离级别（先 `READ COMMITTED`）
- [X] 锁管理（行锁或键范围锁，先简化实现）
- [X] 死锁检测或锁等待超时

### D2. WAL 与恢复

- [X] WAL 结构版本化
- [X] 崩溃恢复一致性增强（更多故障注入测试）
- [X] checkpoint 机制（减少恢复时间）

### D 验收标准

- [X] 并发写入场景无明显脏写/丢失更新
- [X] 人工 crash 后可自动恢复到一致状态

---

## Milestone E：优化器与性能可观测（P2）

### E1. 统计信息

- [X] 表行数、列基数、空值率
- [X] 自动或手动刷新统计信息

### E2. 执行计划

- [X] `EXPLAIN`
- [X] 基于成本的访问路径选择（全表扫描 vs 索引）
- [X] 多条件过滤下的索引选择策略

### E3. 性能工具

- [X] 慢查询日志
- [X] 基础 profiling 指标（耗时、扫描行数）

### E 验收标准

- [X] 用户可解释“为什么慢”
- [X] 典型查询性能有可量化提升
