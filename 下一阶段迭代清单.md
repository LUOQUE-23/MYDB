# RuseDB 下一阶段迭代清单（V1.0.0 迭代）

## 目标

- 在现有 v0.1 基础上，从“可用”提升到“更可靠、可扩展、可维护”。
- 优先补齐核心数据库语义（约束、DDL 演进、并发与可观测性）。

## 里程碑总览

- Milestone A：约束系统（主键/唯一键/外键）
- Milestone B：DDL 演进能力（ALTER/RENAME）
- Milestone C：查询能力增强（更多谓词与 JOIN 语义）
- Milestone D：事务并发与恢复增强
- Milestone E：优化器与性能可观测
- Milestone F：运维能力与生态接入

---

## Milestone A：约束系统（P0）

### A1. PRIMARY KEY

- [x] SQL 语法支持：列级和表级主键定义
- [x] Catalog 存储主键信息
- [x] 插入/更新时校验主键唯一性
- [x] 主键列默认 `NOT NULL`
- [x] 错误码与错误信息统一（如 duplicate key）
- [x] 单元测试 + 集成测试（插入冲突、更新冲突、空值场景）

### A2. UNIQUE

- [x] SQL 语法支持：列级/表级 UNIQUE
- [x] 建立对应唯一索引或唯一性检查路径
- [x] 插入/更新冲突检测
- [ ] 测试覆盖（多列唯一、重复值、事务内冲突）

### A3. FOREIGN KEY

- [x] SQL 语法支持：`FOREIGN KEY ... REFERENCES ...`
- [x] Catalog 持久化外键元数据
- [x] 插入/更新时引用完整性校验
- [x] 删除/更新父表行为（先支持 `RESTRICT`）
- [x] 跨表事务一致性测试

### A 验收标准

- [x] `CREATE TABLE` 可定义 PK/UNIQUE/FK
- [x] 违反约束时返回稳定可读错误
- [x] 重启后约束元数据仍生效

---

## Milestone B：DDL 演进能力（P1）

### B1. ALTER TABLE

- [ ] `ALTER TABLE ... ADD COLUMN`
- [ ] `ALTER TABLE ... DROP COLUMN`（受限规则先行）
- [ ] `ALTER TABLE ... ALTER COLUMN`（后续可扩）

### B2. RENAME

- [ ] `RENAME TABLE`
- [ ] `RENAME COLUMN`
- [ ] 关联元数据同步（索引、约束引用）

### B3. 安全与兼容

- [ ] DDL 事务边界与失败回滚策略
- [ ] 向后兼容测试（旧数据目录升级）

### B 验收标准

- [ ] 基本 schema 演进可在不重建库的前提下完成
- [ ] DDL 异常不破坏 catalog 一致性

---

## Milestone C：查询能力增强（P1）

### C1. 谓词与表达式

- [ ] `IN`
- [ ] `LIKE`
- [ ] `BETWEEN`
- [ ] `IS NULL / IS NOT NULL`

### C2. JOIN 与聚合

- [ ] `LEFT JOIN`
- [ ] `HAVING`
- [ ] 聚合与分组边界场景测试

### C3. 子查询（分阶段）

- [ ] 第一阶段：`IN (SELECT ...)`
- [ ] 第二阶段：标量子查询

### C 验收标准

- [ ] 常见业务 SQL 不需要大量在应用层改写
- [ ] SQL 解析报错位置准确、信息可读

---

## Milestone D：事务并发与恢复增强（P1）

### D1. 并发控制

- [ ] 明确隔离级别（先 `READ COMMITTED`）
- [ ] 锁管理（行锁或键范围锁，先简化实现）
- [ ] 死锁检测或锁等待超时

### D2. WAL 与恢复

- [ ] WAL 结构版本化
- [ ] 崩溃恢复一致性增强（更多故障注入测试）
- [ ] checkpoint 机制（减少恢复时间）

### D 验收标准

- [ ] 并发写入场景无明显脏写/丢失更新
- [ ] 人工 crash 后可自动恢复到一致状态

---

## Milestone E：优化器与性能可观测（P2）

### E1. 统计信息

- [ ] 表行数、列基数、空值率
- [ ] 自动或手动刷新统计信息

### E2. 执行计划

- [ ] `EXPLAIN`
- [ ] 基于成本的访问路径选择（全表扫描 vs 索引）
- [ ] 多条件过滤下的索引选择策略

### E3. 性能工具

- [ ] 慢查询日志
- [ ] 基础 profiling 指标（耗时、扫描行数）

### E 验收标准

- [ ] 用户可解释“为什么慢”
- [ ] 典型查询性能有可量化提升

---

## Milestone F：运维能力与生态接入（P2）

### F1. 备份恢复

- [ ] 在线/离线备份
- [ ] 时间点恢复（PITR，按能力阶段推进）

### F2. 权限系统

- [ ] 用户管理
- [ ] 基础 RBAC（库级、表级）

### F3. 接入体验

- [ ] 错误码文档
- [ ] HTTP API 补充管理接口
- [ ] 迁移工具（migrate up/down）

### F 验收标准

- [ ] 单机部署可支撑日常开发与小规模测试环境
- [ ] 具备最小可运维能力
